# Этап — "Печатает..." + статус онлайн/последний визит

## Что добавлено

**Онлайн-статус:**
- При сворачивании/разворачивании приложения (весь процесс, не отдельный экран) автоматически обновляется `users/{uid}.isOnline` и `lastSeen` в Firestore
- В шапке чата и в профиле собеседника — "в сети" / "был(а) N мин/ч назад"

**Индикатор "печатает...":**
- При вводе текста в поле сообщения — статус пишется в `chats/{chatId}.typingUsers.{uid}`
- Автоматически сбрасывается через 3 секунды без нажатий клавиш, а также сразу при отправке сообщения
- У собеседника вместо "в сети" временно показывается "печатает..." (подсвечено синим)

## Важное упрощение (сознательно, для MVP)

Это **не** "настоящая" enterprise-presence система вроде той, что использует Firebase Realtime Database с `onDisconnect()`. Наша реализация обновляет статус только при явных событиях (открыл/свернул приложение). Если приложение **аварийно завершится** (крашнется) или ОС молча убьёт процесс без вызова `onStop()` — статус может остаться "в сети" до следующего открытия/закрытия приложения.

Настоящая presence-система, которая мгновенно фиксирует обрыв соединения (даже при отключении интернета/разрядке телефона), потребует Firebase Realtime Database — это отдельная, более сложная интеграция. Сказать, если это критично и нужно сделать сейчас — можем добавить отдельным этапом.

**Список чатов** (`ChatListScreen`) пока показывает статичный индикатор "онлайн" (поле `isOnline` в документе чата), не обновляется в реальном времени — это осталось из более раннего этапа. Presence сейчас реализован только для экрана конкретного чата и профиля собеседника. Если нужно — вынесу в отдельную доработку.

## Ничего дополнительно настраивать в Firebase не нужно

Все новые поля (`isOnline`, `lastSeen` в `users`, `typingUsers` в `chats`) укладываются в уже существующие правила безопасности.
