# Большое обновление: настройки, дизайн, ответы, реакции, Google-вход

## 1. Отправка по Enter (вкл/выкл)

Новая настройка в **Настройки → "Отправка по Enter"**. Включено по умолчанию — клавиатура показывает кнопку "Отправить" вместо переноса строки, Enter сразу шлёт сообщение. Выключишь — Enter станет обычным переносом строки (кнопка отправки остаётся, работает всегда).

## 2. Контраст сообщений собеседника — исправлено

**Причина бага:** цвет текста для чужих сообщений был жёстко закодирован (`YodoOnSurface` — светло-серый, всегда одинаковый, не зависящий от темы). На светлом фоне светло-серый текст на белом пузыре — почти невидим.

**Исправление:** теперь используется `MaterialTheme.colorScheme.onSurfaceVariant` — эта тема-зависимая пара автоматически меняется местами со светлой/тёмной темой и всегда обеспечивает контраст.

## 3. Дизайн — приближен к Telegram

- Пузыри сообщений: асимметричные скруглённые углы (один угол более "острый" — там, где был бы хвостик)
- Более компактные отступы в списке чатов
- Убрал стандартную высокую `TopAppBar` в списке чатов — заменил на компактный заголовок

## 4. Уменьшен отступ сверху + название мессенджера

Раньше был обычный Material3 `TopAppBar` с заголовком "Чаты". Теперь — компактный кастомный заголовок **"Yodo Messenger"**, занимает меньше места по высоте.

## 5. Ответ на сообщение

Долгий тап по сообщению → выпадающее меню → **"Ответить"**. Над полем ввода появляется превью цитируемого сообщения (с крестиком для отмены). При отправке ответ прикрепляется и отображается как цитата внутри пузыря.

## 6. Реакции на сообщения

Тот же долгий тап → ряд эмодзи (👍❤️😂😮😢🙏) → тап по эмодзи ставит/убирает реакцию. Под сообщением появляются маленькие "чипы" с эмодзи и счётчиком, если реагировало больше одного человека. Повторный тап по чипу снимает свою реакцию.

## 7. SHA-1 для Google-входа — теперь без компьютера

Раньше я говорил, что для этого нужен `./gradlew signingReport` на компьютере — но у тебя его нет. Решение: **добавил шаг в сборку APK**, который печатает SHA-1 и SHA-256 debug-ключа прямо в лог GitHub Actions.

### Как получить SHA-1

1. Загрузи обновлённый `.github/workflows/build-apk.yml` в репозиторий
2. **Actions → Build Debug APK → Run workflow**
3. Дождись завершения → открой шаг **"Print debug keystore SHA-1 / SHA-256"**
4. Скопируй строки вида:
   ```
   SHA1: XX:XX:XX:...
   SHA256: XX:XX:XX:...
   ```
5. Firebase Console → ⚙️ **Project settings** → прокрути до Android-приложения → **Add fingerprint** → вставь SHA-1 (и отдельно SHA-256, если попросит) → **Save**
6. Подожди минуту-две, затем попробуй "Войти через Google" в приложении снова
