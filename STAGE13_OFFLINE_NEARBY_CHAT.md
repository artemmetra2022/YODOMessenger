# Офлайн-чат без интернета (Bluetooth/Wi-Fi Direct)

## Важно понимать сразу — что это, а что нет

**Это реализовано:** прямая передача сообщений между двумя телефонами, находящимися физически рядом (обычно 10-100 метров), через Bluetooth/Wi-Fi Direct — **совсем без интернета**, без сервера, без Firebase. Основано на официальном **Google Nearby Connections API**.

**Это НЕ реализовано (и не может быть сделано быстро):** настоящий mesh-протокол, как у Bridgefy/FireChat, где сообщение "прыгает" от телефона к телефону через десятки посторонних устройств, покрывая километры. Это отдельный, самостоятельный протокол маршрутизации — у команды Bridgefy на его разработку ушли годы, а сам SDK давно закрыт для новых интеграций. Наша реализация — это честная **прямая P2P-связь** между двумя устройствами в радиусе действия, что уже само по себе решает сценарий "нет сети/интернета, но люди рядом".

## Как пользоваться

1. Открой список чатов → кнопка **"+"** → **"Офлайн-чат (Bluetooth)"**
2. Разреши запрошенные разрешения (Bluetooth + геолокация — она обязательна технически для поиска устройств по Bluetooth на Android, независимо от того, что делает конкретное приложение; сами координаты никуда не передаются)
3. Приложение начинает одновременно **искать** других поблизости и **быть видимым** для них
4. Как только собеседник тоже откроет этот экран на своём телефоне — вы увидите друг друга в списке
5. Тапни на найденное устройство → подключение установится за несколько секунд
6. Пишите сообщения — они идут напрямую, без интернета

## Технические детали

- **Протокол:** Google Nearby Connections, стратегия `P2P_CLUSTER`
- Автоматически выбирает лучший доступный канал связи (Bluetooth Classic, BLE или Wi-Fi Direct) — не нужно ничего настраивать вручную
- Входящие подключения принимаются автоматически (упрощение MVP — без диалога подтверждения "разрешить подключение?"). Это ок для сценария "мы оба открыли приложение специально, чтобы связаться", но менее безопасно, если рядом много посторонних Yodo-пользователей одновременно ищут собеседника — теоретически может подключиться кто угодно поблизости, а не именно тот, кому ты писал. Если важно — добавлю подтверждение по коду/имени перед следующим этапом.
- Сообщения **не сохраняются** между сессиями (простое решение для MVP — история существует только пока открыт экран). Если нужно — можно добавить локальное сохранение (Room, уже подключён в проекте).
- Работает независимо от основного (Firebase) чата — это **отдельный** режим, сообщения из офлайн-чата не появляются в обычном списке чатов и наоборот.

## Ничего в Firebase настраивать не нужно

Этот функционал целиком локальный, не использует Firebase/интернет вообще.

## Новые файлы

- `offline/OfflineModels.kt` — модели данных
- `offline/NearbyMessagingManager.kt` — обёртка над Nearby Connections API
- `offline/OfflineChatViewModel.kt`, `OfflineChatScreen.kt` — UI
- `features/main/MainScreen.kt` — пункт меню "Офлайн-чат"
- `navigation/Routes.kt`, `YodoNavGraph.kt` — маршрут
- `AndroidManifest.xml` — разрешения Bluetooth/Wi-Fi/геолокация
- `app/build.gradle.kts` — зависимость `play-services-nearby`

## Что можно улучшить в будущем (не сделано сейчас)

- Диалог подтверждения входящего подключения (по коду/имени) вместо автопринятия
- Локальное сохранение истории офлайн-переписки (Room)
- Отправка не только текста, но и фото (Nearby Connections поддерживает передачу файлов через `Payload.fromFile()`)
- Индикация уровня сигнала/качества соединения
