# Этап 3 — Телефонный вход + Список чатов: что нужно настроить

## 1. Включить вход по телефону

В Firebase Console → **Authentication → Sign-in method → включи "Phone"**.

**Важно:** для работы SMS на реальных устройствах нужно добавить **SHA-1 и SHA-256** отпечатки твоего приложения в настройки Android-приложения в Firebase (Project settings → твоё приложение → Add fingerprint). Получить их можно командой в Android Studio:
```
./gradlew signingReport
```
Без этого шага Firebase может требовать reCAPTCHA-подтверждение в браузере вместо автоматической SMS-верификации — это ожидаемо для debug-сборок без SHA-ключей, не баг.

## 2. Настроить Firestore

Если ещё не создавал: **Firestore Database → Create database** (test mode для разработки).

### Структура данных, которую ожидает код

Коллекция `chats`, документ на каждый чат:
```json
{
  "participantIds": ["uid1", "uid2"],
  "title": "Иван Петров",
  "avatarUrl": null,
  "type": "PRIVATE",
  "lastMessage": "Привет! Как дела?",
  "lastMessageTimestamp": 1720000000000,
  "unreadCounts": { "uid1": 0, "uid2": 3 },
  "isOnline": true
}
```

### Составной индекс (важно!)

Запрос списка чатов использует `whereArrayContains("participantIds", uid)` + `orderBy("lastMessageTimestamp")` — Firestore потребует **composite index**. При первом запуске приложения открой Logcat, найди ошибку от Firestore — там будет **прямая ссылка**, кликни её, и Firebase Console сам создаст нужный индекс за 1 клик.

### Тестовые данные

Чтобы проверить экран списка чатов прямо сейчас, добавь вручную в Firestore Console пару документов в коллекцию `chats` с `participantIds`, содержащим твой `uid` (посмотреть uid можно в Authentication → Users).

## Что уже работает в коде

- Полный флоу входа по телефону: ввод номера → SMS-код → автоматическая верификация (SMS Retriever) или ручной ввод кода
- Понятные ошибки на русском (неверный код, слишком много попыток и т.д.)
- Список чатов — **реалтайм** (любое изменение в Firestore мгновенно отражается в UI без перезагрузки)
- Аватар-заглушка с первой буквой имени, индикатор "онлайн", бейдж непрочитанных, время последнего сообщения
- Состояния Loading / Empty / Content обработаны
