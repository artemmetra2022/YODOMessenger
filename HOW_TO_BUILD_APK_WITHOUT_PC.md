# Как собрать APK без компьютера — через GitHub Actions

Всё делается с телефона или планшета через браузер. Компьютер не нужен вообще.

## Шаг 1. Создай аккаунт на GitHub

Если ещё нет — зайди на https://github.com и зарегистрируйся (email + пароль).

## Шаг 2. Создай новый репозиторий

1. В приложении GitHub (или через браузер) нажми **+ → New repository**
2. Имя: `yodo-messenger` (любое)
3. Сделай **Private** (чтобы код не был публичным) — важно, так как внутри будут упоминания структуры проекта
4. Нажми **Create repository**

## Шаг 3. Загрузи файлы проекта в репозиторий

Самый простой способ с телефона — через веб-интерфейс GitHub:

1. Открой созданный репозиторий в браузере
2. Нажми **Add file → Upload files**
3. Распакуй присланный тебе zip-архив (`YodoMessenger-stageX.zip`) на телефоне любым файловым менеджером/архиватором
4. Перетащи или выбери **все файлы и папки** изнутри распакованной папки `YodoMessenger` (важно грузить содержимое папки, а не саму папку) и загрузи
5. Нажми **Commit changes**

> Если загрузка большого количества файлов через мобильный браузер неудобна — можно установить приложение **GitHub Mobile** или **Working Copy** (iOS) / **MGit** (Android), они дают более удобную загрузку папок.

## Шаг 4. Получи google-services.json и закодируй его

1. Зайди в Firebase Console → Project settings → скачай `google-services.json` (см. `STAGE2_FIREBASE_SETUP.md`)
2. Файл нужно превратить в **base64-строку** — на телефоне это можно сделать через сайт https://base64.guru/converter/encode/file — загрузи файл, получи текстовую строку, скопируй её целиком

## Шаг 5. Добавь секрет в репозиторий

1. В репозитории на GitHub: **Settings → Secrets and variables → Actions**
2. **New repository secret**
3. Name: `GOOGLE_SERVICES_JSON`
4. Secret: вставь скопированную base64-строку
5. **Add secret**

## Шаг 6. Запусти сборку

1. Открой вкладку **Actions** в репозитории
2. Слева выбери workflow **"Build Debug APK"**
3. Справа нажми **Run workflow → Run workflow**
4. Подожди 3-7 минут — появится зелёная галочка ✅ (если что-то не так — красный крестик ❌, тогда открой лог и пришли мне текст ошибки)

## Шаг 7. Скачай готовый APK

1. Открой завершённый запуск (зелёная галочка)
2. Внизу страницы — раздел **Artifacts**
3. Скачай `yodo-messenger-debug-apk` — это zip с файлом `app-debug.apk` внутри
4. Распакуй, открой `app-debug.apk` на Android-телефоне (нужно разрешить "Установка из неизвестных источников" в настройках) — приложение установится

## Что важно понимать про эту сборку

- Это **debug-сборка** — для тестирования. Она не подписана релизным ключом и не подходит для публикации в Google Play (это отдельный, более сложный шаг с созданием keystore — сделаем, когда дойдём до релиза)
- Каждый раз, когда я присылаю новый zip с обновлённым кодом — нужно **заново загрузить изменённые файлы** тем же способом (шаг 3) и снова запустить workflow (шаг 6)
- Если структура проекта в архиве совпадает с уже загруженной — GitHub веб-интерфейс предложит **перезаписать** существующие файлы при загрузке, это нормально
