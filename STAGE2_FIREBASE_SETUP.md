# Этап 2 — Firebase Auth: что нужно настроить

Код готов, но для реальной работы входа/регистрации нужно подключить твой Firebase-проект.

## Шаги (10 минут)

1. Зайди на https://console.firebase.google.com → **Create project** (или используй существующий).
2. Добавь Android-приложение:
   - Package name: `app.yodo.messenger`
   - Скачай файл **`google-services.json`**
3. Положи `google-services.json` в папку `app/` проекта (рядом с `build.gradle.kts`, на одном уровне с папкой `src`).
4. В консоли Firebase: **Authentication → Sign-in method → включи "Email/Password"**.
5. (Опционально сейчас, понадобится позже) **Firestore Database → Create database** — режим test mode для разработки.
6. Синхронизируй Gradle в Android Studio и запусти приложение.

## Что уже работает в коде

- Регистрация: создаёт пользователя в Firebase Auth + документ в Firestore (`users/{uid}`)
- Вход: проверяет email/пароль через Firebase Auth
- Ошибки переведены на русский (неверный пароль, email занят, нет сети и т.д.)
- При перезапуске приложения — если пользователь уже вошёл, сразу открывается список чатов (пропускает Welcome/Login)
- Валидация на клиенте: email должен содержать `@`, пароль ≥ 6 символов

## Важно

- Поле в макете называется "Телефон или email" — я реализовал через **email**, так как Firebase Phone Auth требует SMS-верификацию и reCAPTCHA — это отдельная, более сложная интеграция. Скажи, если её тоже нужно подключить — сделаю отдельным этапом.
- Без `google-services.json` в папке `app/` **сборка Gradle упадёт с ошибкой**. Это единственный файл, который я не могу сгенерировать за тебя — он привязан к твоему Firebase-аккаунту.
