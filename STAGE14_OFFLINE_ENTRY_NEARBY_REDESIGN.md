# Режим без интернета с главного экрана + "Кто рядом" + редизайн

## 1. Кнопка "Режим без интернета" на Welcome ✅

На стартовом экране теперь две дорожки: обычный вход/регистрация ИЛИ сразу **"Режим без интернета"** → мгновенно открывает офлайн Bluetooth-чат (из прошлого этапа), минуя Firebase полностью — как и должно быть, раз без сети регистрация невозможна.

## 2. "Кто рядом" — список + карта ✅

Реализовано на **геолокации** (не на Bluetooth, это отдельный от офлайн-чата механизм):

- Экран запрашивает геопозицию, публикует её в свой профиль (Firestore), ищет других пользователей, тоже недавно открывавших этот экран (окно 15 минут), в радиусе 10 км
- Расстояние считается на устройстве (формула Haversine), без сторонних гео-индексов
- Две вкладки: **Список** (сортировка по расстоянию) и **Карта** с метками
- При выходе с экрана геопозиция удаляется из профиля — приватность по умолчанию

### Важно про карту — почему не Google Maps

Google Maps SDK для Android **тоже требует привязанный биллинг-аккаунт** даже для бесплатного тарифа — тот же блокер, что был с Firebase Storage и Cloud Functions. Поэтому использовал **OpenStreetMap** (библиотека `osmdroid`) — полностью бесплатно, без API-ключа, без карты.

## 3. Редизайн — честно про масштаб

Референс, который ты прислал, — это набор из **12+ полностью проработанных экранов** (список чатов, звонки, статистика, продвинутые настройки и т.д.), многих из которых у нас в приложении пока просто не существует как функций (статистика использования, отдельная вкладка звонков и т.п.).

**Что я сделал сейчас:**
- Список чатов: аватары теперь с цветным градиентным кольцом вокруг (фиолетово-синий бренд-градиент) для онлайн-статуса — прямо как в референсе
- Обновлена палитра под фирменный градиент

**Что НЕ сделано (нужно уточнить приоритеты)** — полное пиксель-в-пиксель повторение требует по сути переделать почти каждый экран:
- Карточки с "Media"/"Files" в профиле собеседника
- Отдельная вкладка "Calls" с историей звонков (звонков в приложении вообще ещё нет)
- Экран "Statistics" (это не мессенджер-функция, а, похоже, панель разработчика/админа — уточни, точно ли она нужна пользователю)
- Специфичная форма карточек, поисковая строка с закруглением как на макете, боковая структура для планшетов/десктопа (у нас чисто мобильное приложение, макет больше похож на desktop/web-версию)

Предлагаю: скажи, **какие 2-3 экрана важнее всего** визуально довести до этого уровня в первую очередь (например, именно список чатов + карточку чата) — и я сделаю их максимально близко к референсу, вместо распыления на всё сразу.
