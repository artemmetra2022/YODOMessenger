# Диагностика: чат не появляется в главном меню

## Что я сделал

Раньше, если Firestore возвращал ошибку при загрузке списка чатов (например, не хватает индекса или не пускают правила безопасности) — приложение **молча показывало "чатов нет"**, как будто чатов действительно нет. Это маскировало настоящую проблему.

Теперь ошибка **видна на экране** — вместо пустого списка появится текст с точным сообщением Firestore. То же самое для ошибки создания чата через поиск — раньше терялась молча, теперь показывается всплывающим уведомлением (Snackbar) внизу экрана.

## Что нужно от тебя

1. Загрузи обновлённые файлы (список ниже)
2. Пересобери APK
3. Повтори тот же сценарий: напиши человеку через поиск
4. **На стороне получателя** открой вкладку "Чаты" — если баг всё ещё есть, теперь вместо пустого списка должно появиться сообщение с текстом ошибки
5. **Пришли мне точный текст этой ошибки** — по нему я сразу пойму, в чём дело:
   - Если там что-то про `"index"` / `"requires an index"` — значит не хватает Firestore-индекса, там же в тексте (или в консоли) будет ссылка на автосоздание
   - Если про `"permission"` / `"insufficient permissions"` — значит дело в правилах безопасности Firestore (Rules), пришлю обновлённые
   - Если чат по-прежнему просто не появляется, но ошибки нет — значит дело не в загрузке списка, а в чём-то другом, будем смотреть дальше

## Изменённые файлы

- `domain/repository/ChatRepository.kt` — новый тип `ChatListResult` (Success/Error)
- `data/repository/ChatRepositoryImpl.kt` — ошибки логируются и передаются наружу вместо проглатывания
- `features/chats/ChatListViewModel.kt`, `ChatListScreen.kt` — новое состояние Error с текстом
- `features/search/SearchViewModel.kt`, `SearchScreen.kt` — ошибка создания чата теперь показывается Snackbar'ом
