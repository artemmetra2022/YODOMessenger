# Этап 8 — Профиль собеседника: что нового

## Что добавлено

- Новый экран `UserProfileScreen` — просмотр чужого профиля (только чтение): аватар, имя, @username, "о себе"
- Кнопка **"Написать сообщение"** — создаёт (или открывает существующий) приватный чат и сразу переходит в него
- **Заголовок экрана чата** теперь показывает реальное имя собеседника (а не статичное "Чат") и **кликабелен** — тап открывает его профиль
- В поиске тап по **аватару** открывает профиль, тап по всей строке — сразу открывает чат (как раньше)

## Как это работает технически

- `ChatRepository.getChatInfo(chatId)` — при открытии чата один раз подтягивает персонализированное название и id собеседника (для приватных чатов)
- `UserRepository.getUserById(uid)` — разовая загрузка публичного профиля (не realtime-подписка, профиль собеседника не обязан обновляться мгновенно)

## Ничего дополнительно настраивать в Firebase не нужно

Этот этап целиком использует уже существующие коллекции `users` и `chats` — новых индексов или правил не требуется.

## Изменённые/новые файлы

- `domain/repository/UserRepository.kt` + `UserRepositoryImpl.kt` — метод `getUserById`
- `domain/repository/ChatRepository.kt` + `ChatRepositoryImpl.kt` — модель `ChatInfo`, метод `getChatInfo`
- `features/profile/UserProfileViewModel.kt`, `UserProfileScreen.kt` — новые
- `features/chats/ChatViewModel.kt` — добавлены `chatTitle`, `otherUserId`
- `features/chats/ChatScreen.kt` — кликабельный заголовок
- `features/search/SearchScreen.kt` — клик по аватару открывает профиль
- `navigation/Routes.kt`, `YodoNavGraph.kt` — маршрут `UserProfile`

- Нет блокировки/жалобы на пользователя — это можно добавить отдельным пунктом позже
