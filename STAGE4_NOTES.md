# Этап 4 — Экран личного чата: что нужно знать

## Что работает

- **Реалтайм-сообщения**: подписка на `chats/{chatId}/messages`, сортировка по времени — новое сообщение появляется у собеседника мгновенно, без перезагрузки экрана
- **Отправка**: при отправке сообщение пишется в подколлекцию + одновременно обновляется родительский документ чата (`lastMessage`, `lastMessageTimestamp`) и увеличивается счётчик непрочитанных **у всех остальных участников** — список чатов (Этап 3) обновится автоматически
- **markChatAsRead**: при открытии чата счётчик непрочитанных **для тебя** сбрасывается в 0
- Пузыри сообщений: свои — справа синим, чужие — слева серым, с временем отправки
- Автоскролл к последнему сообщению
- Пустое состояние ("Сообщений пока нет")

## Что важно проверить в Firestore Rules

Сейчас, скорее всего, у тебя test mode (все могут всё). Для продакшена понадобятся правила безопасности — например:

```
match /chats/{chatId} {
  allow read, update: if request.auth.uid in resource.data.participantIds;
  match /messages/{messageId} {
    allow read, create: if request.auth.uid in get(/databases/$(database)/documents/chats/$(chatId)).data.participantIds;
  }
}
```

Это можно настроить позже — не блокирует разработку.
