# Этап 9 — Статус прочтения, групповые чаты, нижнее меню

## 1. Статус "прочитано" ✅

- При открытии чата все чужие сообщения помечаются `status: "READ"` батчем в Firestore
- В пузыре своих сообщений теперь показана иконка:
  - ✓ (одна галочка, серая) — отправлено
  - ✓✓ (две галочки, голубые) — прочитано собеседником
- Работает и в группах: сообщение станет "прочитано", как только **хотя бы один** участник откроет чат (это стандартное упрощение для MVP — полноценные "прочитано всеми N из M" потребуют более сложной модели данных)

## 2. Групповые чаты ✅

- Кнопка **"+"** (плавающая, внизу справа на вкладке "Чаты") → меню "Новый чат" / "Новая группа"
- Экран создания группы: название + поиск и мультивыбор участников (минимум 2 человека + ты)
- Групповой чат использует то же самое окно чата (`ChatScreen`), что и личные — сообщения, статусы и т.д. работают одинаково
- В списке чатов групповой чат отображается со своим статичным названием (в отличие от личных чатов, где у каждого — имя собеседника)

## 3. Нижнее меню в стиле Telegram ✅

- Новый `MainScreen` — контейнер с `NavigationBar` внизу: **Чаты** / **Настройки**
- Список чатов и настройки теперь переключаются по нижним вкладкам, а не отдельными экранами с "назад"
- Профиль, поиск, создание группы и сам чат остаются полноэкранными (открываются поверх, с кнопкой "назад") — так же было в Telegram

## Ничего дополнительно настраивать в Firebase не нужно

Использует существующие коллекции. Единственный нюанс: если после создания группы Firestore попросит новый индекс (для запросов с `whereNotEqualTo` в статусах прочтения) — как обычно, в логах будет ссылка для автосоздания в один клик.
